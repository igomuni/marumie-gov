# 予算0で支出がある事業の調査報告

**調査日**: 2025年11月18日
**対象年度**: 2024年度
**調査者**: Claude Code

---

## 概要

2024年度の行政事業レビューデータにおいて、**予算額が0円だが支出額が発生している事業が878件**存在することが判明しました。本レポートでは、この現象の原因と根拠となるデータについて調査結果を報告します。

---

## 調査結果サマリー

- **該当事業数**: 878事業（全5,664事業中、約15.5%）
- **主な原因**: 前年度事業の繰越支出
- **根拠データ**: CSV「備考」フィールドに明記

---

## 原因の特定

### 1. 事業区分: 「前年度事業」

調査した該当事業はすべて**「前年度事業」**として分類されています。

### 2. 2024年度の予算編成変更

**デジタル庁の例**（予算事業ID: 11-15など）:

CSVファイル `1-2_2024_基本情報_事業概要等.csv` の「備考」フィールドに以下の記載：

> ※2023年度までは、デジタル庁一括計上予算として、レビューシート「情報システムの整備(情報通信技術調達等適正・効率化推進費(予算事業ID:0004)」を作成しておりましたが、**2024年度から、個々のシステムごとのシートを作成しています。**

つまり：
- **2023年度まで**: 一括予算として計上（予算事業ID: 0004）
- **2024年度から**: 個別システムごとにシート分割
- **結果**: 新規に作成されたシート（ID: 11-15等）は2024年度予算0円だが、2023年度以前の支出実績を持つ

## データ構造の問題点

### 1. 年度の不整合

- **CSVファイル名**: `year_2024`（2024年度を示唆）
- **実際のデータ**: 2023年度の執行額
- **予算年度フィールド**: 複数年度のデータが混在

### 2. 前処理での年度フィルタリング

現在の前処理スクリプト（`scripts/preprocess-data.ts`）では：

```typescript
// 対象年度のデータのみをフィルター
const currentYearBudgetData = budgetData.filter((budget) => budget.予算年度 === year);
```

この処理により：
- **2024年度予算**: 0円 → ✅ 正しく取得
- **2023年度支出**: 非0円 → ❌ フィルタで除外される可能性

しかし、`5-1_支出情報.csv` では「事業年度」フィールドが2024として記録されているため、支出データは取得される。

## 結論

### 主要な原因

**予算0で支出がある事業の大半は、前年度事業の執行実績が2024年度データとして記録されているため**です。

### 具体的なケース

1. **予算編成変更**: 一括予算から個別予算への変更（デジタル庁の例）
2. **事業終了**: 2024年度で終了予定の事業が、過去の執行実績のみを持つ
3. **繰越執行**: 前年度からの繰越予算による支出（予算年度と執行年度のずれ）

### データの信頼性

CSVの「備考」フィールドに詳細な説明があり、データの妥当性は確認できます。ただし、**年度の概念が複雑**（予算年度・事業年度・執行年度）であるため、ユーザーには注意深い説明が必要です。

## 推奨事項

### 1. ユーザー向け説明の追加

予算と支出詳細モーダルに、以下の注記を追加することを推奨：

> ℹ️ **注意**: 予算が0円で支出がある事業は、前年度事業の執行実績や予算編成変更により発生する場合があります。詳細は各事業の「備考」をご確認ください。

### 2. 事業詳細での「備考」表示

事業をクリックした際のモーダルで、CSV「備考」フィールドを表示し、背景情報を提供する。

### 3. フィルタリングオプション

「前年度事業を除外」オプションを追加し、現年度予算のみを表示できるようにする。
